# transformaciones-transiciones-css

Transforma y anima elementos con CSS. Aprende a usar transformaciones 2D y 3D, transiciones y efectos como Parallax. Entiende conceptos de UX y optimiza animaciones para rendimiento y accesibilidad, creando interfaces más dinámicas.

## 📚 Contenido del Curso y Aprendizajes

A lo largo de este curso, se exploraron y dominaron los siguientes conceptos clave a través de 20 lecciones prácticas divididas en módulos estratégicos:

### 1. Importancia de las Animaciones Web

- **Conceptos iniciales:** Beneficios y aplicaciones de las transiciones y transformaciones en interfaces modernas.
- **Propiedades animables:** Introducción a las bases de `transform` y `transition`.
- **Interactividad:** Uso avanzado de pseudoclases y pseudoelementos.
- **Curvas de tiempo:** Funciones de temporización y manejo de planos espaciales.

### 2. Transformaciones en 2D y 3D

- **Propiedad Transform:** Implementación y diferencias de movimiento en ejes 2D y 3D.
- **Herramientas de distorsión:** Uso y combinación de `scale()`, `skew()` y `rotate()`.
- **Puntos de anclaje:** Modificación del eje de rotación con `transform-origin`.
- **Profundidad y visibilidad:** Control tridimensional y volteo de elementos con `backface-visibility`.

### 3. Parallax con Transformaciones

- **Efecto de profundidad:** Implementación y estilización de efectos Parallax puros utilizando únicamente HTML y CSS estructural.

### 4. Transiciones Avanzadas

- **Fundamentos:** Control milimétrico con `transition-property` y `transition-duration`.
- **Comportamiento dinámico:** Ajustes finos empleando `transition-timing-function` y desfases con `transition-delay`.

### 5. Tips de UX (Experiencia de Usuario)

- **Diseño UI interactivo:** Creación de transiciones de movimiento naturales.
- **Tiempos de espera:** Optimización del flujo y retardo en menús desplegables para evitar cierres accidentales.
- **QA visual:** Solución de parpadeos (_flickering_) molestos durante los estados `:hover`.

### 6. Rendimiento y Accesibilidad

- **Animaciones eficientes:** Selección de propiedades óptimas que no gatillen _layout_ o _paint_ innecesarios (enfocándose en opacidad y transformaciones).
- **Optimización de hardware:** Uso correcto de la aceleración por hardware y la propiedad `will-change`.
- **Inclusión:** Respeto a la accesibilidad web adaptando la interfaz mediante la media query `@media (prefers-reduced-motion)`.
